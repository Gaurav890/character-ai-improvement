Where it isn't landing yet

Four specific gaps, not a general complaint

1. Four systems, four mental models

Story Memory, Facts, the meter, and Lorebook each work differently and live in different places. Nobody outside your team is expected to track that — and it shows in reviews that describe memory as unpredictable rather than describing a specific broken feature.

2. The trust-building pieces are all paywalled

Facts, the meter, and first Lorebook access are c.ai+ only. Free users — who write most of the negative reviews — can't even see what the Character currently remembers, let alone fix it.

3. When it forgets, it guesses instead of saying so

Independent testing found the model fills memory gaps with confident, invented detail rather than flagging uncertainty. That quietly erodes trust in ways that feel worse than forgetting outright.

4. The paid promise isn't holding up

People are already paying for c.ai+ expecting better memory and saying it isn't there. That's the most fixable and most urgent item on this list — it's an existing customer commitment, not a hypothetical feature.

What I'd suggest

Five concrete changes, in rough priority order

Design

Merge Story Memory, Facts, and Lorebook into one "Memory & Canon" panel

One place, organized by Persona · Relationship History · World Rules · Key Events — instead of four features a user has to learn separately. See the concept sketch below.

Access

Give free users read + basic edit access to that panel

Keep deeper capacity (more pins, auto-Facts extraction) behind c.ai+ if you need the tier distinction — but let every user at least see and correct what's remembered about their own persona. That's where most of the complaint volume is coming from.

Trust

Show what the model actually used this turn

A small, dismissible strip — "used: [fact], [fact] · dropped: [fact]" — turns invisible failure into a visible, fixable one. This targets the confident-fabrication problem directly, which storage alone doesn't solve.

Trust

Close the c.ai+ memory gap before promoting it further

Fix or clearly re-scope what the memory upgrade actually delivers for paying users. This is the cheapest reputational win on the list — it's about honoring an existing promise, not building something new.

Measurement

Track memory-feature usage against retention and ratings directly

Public reviews can't show whether Facts/Lorebook already helped — only your own data can. Worth confirming before investing further, so the next round is judged on real numbers instead of public sentiment alone.
