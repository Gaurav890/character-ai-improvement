One feature I'd be excited to add or improve in Character.AI: a memory & canon layer users can see, pin, edit, and trust across chats

Memory is the #1 thing your own users ask for — you've said so yourselves. Since early 2025 you've shipped Story Memory, Facts, and Memory Usage, and as of the May 21, 2026 update, folded them into one redesigned Memory screen. Lorebook shipped in April 2026 alongside it, with the two explicitly planned to work together. That's real, fast progress.

But public sentiment on memory hasn't moved with it — ratings are still sitting around 1.3–1.5★ in reviews that name memory specifically, and some users who upgraded to c.ai+ for it say it isn't any better than free.

My read: Lorebook is still the one piece living outside the unified Memory screen, the parts of that screen that would actually build trust (Facts, full usage visibility) are paywalled, and the model doesn't fail loudly enough when it forgets — it fills gaps with confident guesses instead of telling the user what dropped. Six suggestions below aim at exactly those three things.

03Where it isn't landing yet

1. Lorebook is the one piece still living outside the unified screen

Story Memory, Facts, and Memory Usage are already one screen — genuinely good progress. Lorebook shipped separately the same spring and, per your own post, is only planned to "work alongside" that screen, not inside it yet. That's the one seam left to close.

2. The parts of that screen that build trust are still paywalled

Free users can see Story Memory and a simplified usage view, but not Facts or the full picture of what's stored. They can see that something is remembered but not exactly what, or fix it if it's wrong — and they're the ones writing most of the negative reviews.

3. When it forgets, it guesses instead of saying so

Your own Challenges and Limitations page says it plainly: characters "can confidently tell you things that are verifiably false." Independent testing found the same pattern in practice — the model fills memory gaps with invented detail rather than flagging uncertainty. That's already an acknowledged, named limitation, not a hypothesis.

4. The paid promise isn't holding up

People are already paying for c.ai+ expecting better memory and saying it isn't there. That's the most fixable and most urgent item on this list — an existing customer commitment, not a hypothetical feature.

5. Your own creator guide doesn't mention any of this

I read all 40 pages of book.character.ai, your creator-facing documentation. Memory, Facts, Story Memory, and Lorebook aren't mentioned once — not in Character Attributes, Definition, Training, or User Personas. Creators have no official guidance on how these features interact with what they've written.

04What I'd suggest

1. Finish the merge — fold Lorebook into the Memory screen you already redesigned

You've done the hard part: Story Memory, Facts, and Memory Usage already live in one screen. Your own post says Lorebook will "work alongside" it — I'd push for "live inside" instead, so it's one place to check rather than one more surface to remember exists. Concept sketch below (caveats included — I haven't seen the actual screen).

2. Give free users at least read access to Facts

Free users can already see Story Memory and a simplified usage view — that's good. But Facts (the structured, auto-extracted view of what the Character actually knows) is c.ai+ only, so free users still can't see or correct the part most likely to be wrong. Keep auto-extraction depth or edit limits behind c.ai+ if you need the tier distinction, but let everyone at least see it.

3. Show what the model actually used this turn

A small, dismissible strip — "used: [fact], [fact] · dropped: [fact]" — turns invisible failure into a visible, fixable one. Targets the confident-fabrication problem directly, which storage alone doesn't solve.

4. Close the c.ai+ memory gap before promoting it further

Fix or clearly re-scope what the memory upgrade actually delivers for paying users. The cheapest reputational win on this list — honoring an existing promise, not building something new.

5. Track memory-feature usage against retention and ratings directly

Public reviews can't show whether Facts/Lorebook already helped — only your own data can. Worth confirming before investing further, so the next round is judged on real numbers instead of public sentiment alone.

6. Add Memory, Facts, and Lorebook to the creator guide

The cheapest item on the list. book.character.ai walks creators through Definition, Training, and User Personas in detail but never mentions the memory tools shipped this spring. Creators can't design around canon consistency features they don't know exist.
